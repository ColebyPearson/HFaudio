==========
UNIT 6: FROM TEXT TO SPEECH
==========

--- TTS Datasets ---

Text-to-speech presents unique challenges compared to other audio tasks. TTS involves a one-to-many mapping problem, meaning the same text can be spoken in many different ways depending on intonation, emphasis, pacing, and speaker characteristics. There are also long-distance dependencies to manage, and the task requires paired text-speech data with high quality recordings. ASR datasets are generally not suitable for TTS because they tend to have background noise and other artifacts that would be reproduced by a trained model.

Key datasets used in TTS research and development:

LJSpeech: Contains 13,100 English audio clips from a single speaker. It is high quality and commonly used as a benchmark for TTS systems.

Multilingual LibriSpeech: Covers multiple languages and is derived from audiobooks, providing natural reading-style speech.

VCTK: Features 110 English speakers with various accents, each reading approximately 400 sentences. This diversity makes it valuable for multi-speaker TTS.

Libri-TTS and LibriTTS-R: Provides 585 hours of audio at 24kHz with multiple speakers. LibriTTS-R is a cleaned version of LibriSpeech specifically prepared for TTS use cases.


--- Pre-trained TTS Models ---

SpeechT5 is a unified model designed for multiple speech tasks. It uses an encoder-decoder transformer architecture with modal-specific pre-nets and post-nets. The model is first pre-trained on unlabeled speech and text data, then fine-tuned for specific tasks.

For TTS specifically, SpeechT5 uses a text encoder pre-net consisting of a text embedding layer, a speech decoder pre-net that compresses the log-mel spectrogram via linear layers, and a speech decoder post-net that refines the output spectrogram to improve quality.

SpeechT5 requires speaker embeddings, specifically x-vectors, to control voice characteristics. The output of the model is a log-mel spectrogram, which needs a vocoder such as HiFi-GAN to convert it into an audible waveform.

HiFi-GAN vocoder is a generative adversarial network with a generator and two discriminators. The generator converts mel-spectrograms to raw audio waveforms. It is trained adversarially to produce high-fidelity audio that sounds natural.

Using SpeechT5 in code involves loading the processor, model, and vocoder:
- SpeechT5Processor from "microsoft/speecht5_tts" handles text tokenization
- SpeechT5ForTextToSpeech from "microsoft/speecht5_tts" is the main model
- SpeechT5HifiGan from "microsoft/speecht5_hifigan" converts spectrograms to waveforms
- Speech is generated by calling model.generate_speech with input_ids, speaker_embeddings, and the vocoder

Bark is a transformer-based model by Suno AI that generates raw waveforms directly without needing a separate vocoder. It is made of 4 sub-models and uses Encodec for audio compression. Bark supports multilingual speech, non-verbal sounds like laughter and sighing, and even music generation. It is very controllable through the use of voice presets.

MMS, which stands for Massive Multilingual Speech, is based on the VITS architecture. It supports over 1,100 languages, making it the most linguistically diverse TTS option. MMS generates waveforms directly and uses VitsModel and VitsTokenizer in the HuggingFace ecosystem.


--- Fine-tuning SpeechT5 ---

Fine-tuning SpeechT5 is demonstrated using the VoxPopuli Dutch subset, which contains approximately 20,000 examples from European Parliament recordings.

Preprocessing presents several challenges. The SpeechT5 tokenizer only supports English characters, so special characters from other languages need to be mapped to their English equivalents. For example, accented characters like a-grave are mapped to plain "a" and c-cedilla is mapped to plain "c". Speaker embeddings, specifically x-vectors, must be computed using a speaker verification model such as speechbrain/spkrec-xvect-voxceleb. The dataset should be filtered to include only speakers with between 100 and 400 examples for balance. Long examples exceeding 200 tokens should also be filtered out to avoid memory issues during training.

Speaker embeddings are created using SpeechBrain's x-vector model. Each audio sample receives a 512-dimensional speaker embedding that captures the unique voice characteristics of the speaker.

A custom data collator is needed to pad inputs and labels, replace padding tokens with -100, round target lengths to multiples of the reduction factor, and include speaker embeddings alongside the audio and text data.

Training configuration: 4000 steps, batch size of 4, gradient accumulation steps of 8, learning rate of 1e-5, gradient checkpointing enabled, and fp16 mixed precision. Training takes several hours to complete.

For inference, the fine-tuned model is loaded, a speaker embedding from the dataset is used, and speech is generated with the vocoder.


--- Evaluating TTS Models ---

TTS evaluation is extremely difficult because the quality of speech is inherently subjective. The standard evaluation method is Mean Opinion Score, known as MOS, where human listeners rate speech quality on a scale from 1 to 5. Objective metrics like mean squared error or mean absolute error loss on spectrograms are not very meaningful because many valid spectrograms can accurately represent the same text spoken in different ways.

==========
UNIT 7: PUTTING IT ALL TOGETHER
==========

--- Speech-to-Speech Translation ---

Speech-to-speech translation uses a cascaded approach that combines speech translation with text-to-speech synthesis.

The pipeline works as follows: Input speech in language X is processed by Whisper using the translate task, which converts it to target language text. A TTS model then synthesizes that text into target language speech, producing the output audio.

For the translation component, Whisper with task set to "translate" converts speech directly to English text. For non-English target languages, an additional text translation step is needed. For example, Helsinki-NLP/opus-mt-en-nl can translate English to Dutch.

For the TTS component, SpeechT5 or MMS TTS can be used to generate speech in the target language.

The full pipeline translates the audio to text, synthesizes the text to speech, and converts the output to int16 format for playback. This can be deployed as a Gradio demo with microphone and file upload inputs.


--- Creating a Voice Assistant (Marvin) ---

The voice assistant called Marvin uses a four-stage pipeline.

Stage 1 is Wake Word Detection. An audio classification model, specifically AST fine-tuned on Speech Commands, constantly listens for the wake word "marvin." This is a small model that runs on-device without draining the battery. It uses ffmpeg_microphone_live for real-time audio processing from the microphone.

Stage 2 is Speech Transcription. Whisper Base English provides near real-time transcription. It uses streaming with configurable chunk sizes that allow tuning the tradeoff between speed and accuracy.

Stage 3 is Language Model Query. The transcribed text is sent to a large language model via the HuggingFace Inference API. This stage is fast because only small text tokens are transmitted over the network.

Stage 4 is Speech Synthesis. SpeechT5 TTS generates the spoken response that the user hears.

The full pipeline chains these stages together: launch_fn detects the wake word, transcribe converts the spoken query to text, query sends the text to the language model and gets a response, and synthesise converts that response back to speech.


--- Meeting Transcription ---

Meeting transcription combines automatic speech recognition with speaker diarization, which is the process of identifying who spoke when. Whisper is used for the transcription component and pyannote is used for speaker segmentation. Together they produce a transcript that attributes each segment of speech to the correct speaker.
